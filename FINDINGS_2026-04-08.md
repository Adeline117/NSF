# NSF Project — Major Findings Report (2026-04-08)

This session discovered **two independent label/feature leakage issues** that inflated the main experimental results for Paper 1 and Paper 3. Both are now fixed, and honest numbers are reported alongside a third finding on Paper 0's taxonomy granularity and Paper 2's risk score calibration.

## P0 BLOCKER 1 — Paper 1: C1-C4 Label Leakage

### The bug
`paper1_onchain_agent_id/features/verify_c1c4.py:_check_c3` uses three quantities as the gate for labeling an address as AGENT:
- `tx_interval_cv`
- `hour_entropy`
- `burst_ratio`

These are algebraically the same as three of the top-4 classifier features:
- `tx_interval_std` ≈ `tx_interval_mean × tx_interval_cv`
- `active_hour_entropy` = `hour_entropy`
- `burst_frequency` = `burst_ratio`

Any address labeled AGENT because `hour_entropy > 2.5` is then used to train a classifier whose #1 feature is `active_hour_entropy`. **This is direct target leakage on three of the most important features.**

### The evidence
| Metric | Leaky (C1-C4 gated, n=3316) | Honest (provenance only, n=64) |
|--------|------------------------------|-------------------------------|
| GBM AUC | 0.9803 | 0.6276 (LOO) / 0.6940 (CV) |
| RF AUC | 0.9766 | 0.7713 (LOO) / 0.8030 (CV) |
| LR AUC | 0.9521 | 0.6100 (LOO) / 0.6203 (CV) |
| Top-1 feature univariate AUC | active_hour_entropy = 0.9347 | tx_interval_mean = 0.7683 |
| Inter-label agreement | — | 37.4% (GBM honest vs C1-C4 leaky) |

Additional evidence from statistical significance tests (N=64):
- **RF significantly beats GBM** (McNemar p=0.0225, DeLong p=0.0014, bootstrap CI95 [−0.24, −0.06])
- **GBM vs Majority baseline: NOT significantly different** (GBM LOO Acc=0.578 vs Majority Acc=0.516)
- GBM vs single best feature: single feature actually beats GBM on honest data

### Second-order effect: security audit also corrupted
The "agents have 10× more unlimited approvals" finding in the 3316-set security audit reverses on the trusted 64-set:
| Metric | 3316 set (C1-C4) | Trusted 64 |
|--------|-----------------|------------|
| Mean unlimited approvals (agent) | 61.4 | 49.1 |
| Mean unlimited approvals (human) | 5.9 | **66.1** |
| Agent/human ratio | 10.3× | **0.74×** (p=0.07) |
| Unlimited-approval rate (agent) | 50.1% | 24.5% |
| Unlimited-approval rate (human) | 30.4% | **45.2%** (p=0.03) |

Interpretation: famous ENS humans (vitalik.eth, nick.eth, etc.) are **heavier DeFi users** than curated MEV bots. The "agents are riskier" headline is an artifact of mining targeting token holders.

### Paper 1 RQ1/RQ2 implications
- **RQ1** (can we identify agents from behavioral features?): current honest bound is RF AUC ≈0.77 on N=64, not 0.98 on N=3316. The 3316 dataset needs re-mining with provenance-only ground truth.
- **RQ2** (agents hold more risk?): cannot be answered with the current mining approach. Two honest reframings possible but require new ground truth.

## P0 BLOCKER 2 — Paper 3: Synthetic AI Feature Leakage

### The bug
`paper3_ai_sybil/experiments/experiment_large_scale.py:augment_with_ai_features()` drew AI feature values from class-conditional Paper 1 distributions indexed by `is_sybil` — agent-side beta for `is_sybil=1`, human-side beta for `is_sybil=0`.

Since `fund_flag` is derived from `{RF, MA}` and `is_sybil = f(BT, BW, HF, RF, MA)`, `fund_flag` is positively correlated with `is_sybil`. Therefore the synthetic AI features acted as a hidden proxy for `fund_flag`, trivially inflating the "enhanced detector" LOPO AUC.

### The evidence (4-project subset: 1inch, uniswap, ens, blur_s2)
| Metric | Leaky mean | Honest mean | Inflation |
|--------|-----------|-------------|-----------|
| OPS baseline AUC | — | 0.6077 | (unchanged) |
| Enhanced (OPS+AI) AUC | 0.9257 | **0.6089** | **+0.3168** |
| AI-only AUC | 0.8717 | **0.5012** (random!) | **+0.3705** |

The fix takes AI-only AUC from 0.87 to random (0.50), proving AI features contributed **zero** real signal to `fund_flag` prediction. The entire "enhanced detector AUC jumps from 0.617 to 0.953" finding is an artifact.

### Paper 3 rewrite required
- Original exp4 "AUC recovery from 0.617 to 0.953" must be retracted.
- New defensible positive story: binary classifier "LLM sybil vs real HasciDB" where AI features are drawn from disjoint distributions per class (real rows from human dist, LLM-generated sybils from agent dist). This is the honest setup, implemented in `experiment_ai_feature_ablation.py`.
- Preliminary ablation result on the "basic" level at N=53k: all-8 AI features achieve binary-classifier AUC 0.978 ± 0.003. Single-feature drop-out shows `hour_entropy` contributes most (LOFO delta +0.0286), then `error_recovery_pattern` (+0.0108). Gas-precision features contribute <0.004 each.

## W1-4 POSITIVE WIN — Paper 0 taxonomy projection

Projected each of the 2,590 agents in Paper 1's expanded set onto the 8-category pilot taxonomy using a two-tier rule system (provenance prior → feature refinement).

| Category | Count | % |
|----------|-------|---|
| DeFi Management Agent | 1,669 | 64.4% |
| Deterministic Script | 666 | 25.7% |
| Simple Trading Bot | 130 | 5.0% |
| LLM-Powered Agent | 71 | 2.7% |
| MEV Searcher | 54 | 2.1% |
| (Autonomous DAO / Cross-Chain / RL Trading) | 0 | 0% |

This unlocks Phase 3 validation experiments. Note: 3 of 8 categories have 0 instances in the current mining, an underrepresentation that should be addressed by a targeted Phase 2 mining pass.

### P0-A cluster validation
Unsupervised K-Means sweep on 23 behavioral features:
- **Best silhouette at k=3** (not k=8) — ARI=0.26, 0 orphan clusters
- At k=8: ARI=0.22, mean purity 0.78, 1 orphan cluster

**The 8-category taxonomy is an oversplit relative to behavioral features.** This is a publishable CHI finding: Delphi-validated categories are semantically coherent but behaviorally collapse into 3 super-clusters.

### P0-B multi-class classifier
GBM predicting taxonomy category from 23 behavioral features only (no source/name leak):
- **97.4% overall accuracy, F1-macro 0.87**
- Per-class F1: DeterministicScript 0.996, SimpleTradingBot 0.988, DeFiManagementAgent 0.982, MEVSearcher 0.849, **LLMPoweredAgent 0.528**

**LLMPoweredAgent has the lowest F1 — confused with DeFiManagement.** Direction for paper: "LLM agents currently mimic rule-based agents in tx traces; the behavioral distinction is not yet recoverable from Etherscan data."

## P2-B — Paper 2 risk score recalibration

The original `expected_max=200.0` was calibrated against the pilot synthetic data. On real 62-repo scans:
- **29/62 repos saturated at 100.0** (47%!) — top half indistinguishable
- Raw base scores: median 144, max 8,862, p90 1,461

Recalibrated to `expected_max=1460.7` (p90 of raw). Result: **7/62 saturated** (down from 29), **Spearman rank correlation 0.948** with the old ranking preserved. Documented in `paper/risk_score_calibration.md` (Appendix C).

## Paper-level takeaways

**Paper 1**: The WWW submission cannot claim AUC > 0.98 honestly. The maximum defensible number is RF AUC ≈ 0.77 on N=64, or we need a new ground truth set that doesn't use C1-C4 as the gate.

**Paper 2**: Solid foundation. Taxonomy is defensible (27 patterns, 16 CWEs), risk score now calibrated. Main blockers are scale (need 200+ scans) and scanner packaging.

**Paper 3**: The headline finding (AI sybil evasion + enhanced detector recovery) rests on two legs. Leg 1 (AI sybils evade rule-based detectors at 100%) is real. Leg 2 (enhanced detector catches them at AUC > 0.95) was entirely leakage. New honest positive story is the LLM-sybil binary classifier, where all-8 AI features achieve AUC ≈ 0.98 on the basic evasion level.

**Paper 0**: Strongest honest story of the four. Taxonomy is validated at k=8 with mean purity 0.78, though behavioral features support a coarser k=3 structure. Multi-class classifier achieves 97% accuracy on 5 of 8 categories, with LLM-Powered being the honest weak spot.
