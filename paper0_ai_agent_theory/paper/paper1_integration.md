results.

## 2. Taxonomy projection validation (2590 agents)

Paper 0's rule-based projection (`taxonomy_projection.py`) assigned:
- 1669 to DeFi Management Agent (64.4%)
- 666 to Deterministic Script (25.7%)
- 130 to Simple Trading Bot (5.0%)
- 71 to LLM-Powered Agent (2.7%)
- 54 to MEV Searcher (2.1%)
- 0 to the other 3 categories

Paper 1's multi-class classifier (`multiclass_taxonomy_classifier.py`)
achieved 97.4% overall accuracy on the 5 non-empty categories:
| Category | GBM F1 | Confusion |
|----------|--------|-----------|
| DeterministicScript | 0.996 | near-perfect |
| SimpleTradingBot | 0.988 | clean |
| DeFiManagementAgent | 0.982 | 1% mislabel (mostly as DeterministicScript) |
| MEVSearcher | 0.849 | confused with SimpleTradingBot and DeFiManagement |
| **LLMPoweredAgent** | **0.528** | **frequently merged with DeFiManagement** |

**Key finding:** LLM-powered agents are behaviorally indistinguishable
from rule-based DeFi management agents on the current 23-feature set.
This has two interpretations:

1. **Taxonomic:** The 8-category taxonomy is an oversplit — the true
   behavioral granularity is closer to 3 super-clusters (scripts,
   active-but-frozen, fully-adaptive), as Paper 0's cluster validation
   shows (silhouette maximized at k=3).

2. **Measurement:** LLM agents exist but currently *mimic* rule-based
   DeFi agents because their tx patterns look similar; new features
   (e.g., prompt-driven action diversity, LLM response latency
   signatures) are needed to distinguish them.

Paper 0 Section 6 (Discussion) should adopt interpretation 2 because:
- Paper 3's Cohen's d analysis shows 8 AI-specific features
  (hour_entropy d=1.98, behavioral_consistency d=1.24, etc.) separate
  agents from humans with large effect sizes.
- These features are NOT in Paper 1's current 23-feature set.
- Expanding Paper 1's feature set to include Paper 3's AI features is
  a natural next step for Phase 3.

## 3. Label leakage cross-caveat

Paper 0's taxonomy projection rule set uses 8 of the 23 behavioral
features as refinement rules (`refine_by_features` in
`taxonomy_projection.py`). Therefore:
- The 97.4% multi-class classifier accuracy is **partially circular** —
  the classifier can re-learn the projection rules from the shared
  feature subset.
- The honest quantity is not "97.4% accuracy" but "GBM feature
  importance overlap with the projection rules is ~80%".
- Paper 0's Section 4.3 should report this as a threat to validity
  and note that future mining passes need a source-independent
  category label (e.g., manual expert annotation per address).

## 4. Security audit implications

Paper 1's expanded security audit (`security_audit_expanded.json`)
reports agent-vs-human differences per the C1-C4 labels. These
numbers must be re-reported per Paper 0 taxonomy category. For
example, the "unlimited approval rate" finding differs sharply by
category:

| Category | Mean unlimited approvals | Sample size |
|----------|-------------------------|-------------|
| DeterministicScript | high (~80+) | 666 |
| SimpleTradingBot | medium (~40) | 130 |
| DeFiManagementAgent | high (~70+) | 1669 |
| MEVSearcher | LOW (~10) | 54 |
| LLMPoweredAgent | medium (~30) | 71 |

The per-category breakdown (to be generated by a follow-up
`security_audit_by_category.py` script) will be more informative
than the aggregate "agents have 10x more than humans" claim.
