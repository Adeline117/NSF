# AI Agent 形式化定义（NSF 项目共享基础）

本文件定义了整个 NSF 研究项目中"AI Agent"的形式化标准。
Paper 0 建立定义，Papers 1-3 直接引用。

---

## 1. 核心定义

**定义 1（链上AI Agent）。** 一个链上 AI Agent 是一个满足以下**全部四个必要条件**的软件实体：

> **(C1) 链上执行能力（On-chain Actuation）：** 该实体直接或间接控制一个外部拥有账户（EOA），能够在区块链上签署和提交交易。
>
> **(C2) 环境感知（Environmental Perception）：** 该实体持续或事件驱动地感知链上状态（如价格、余额、事件日志）和/或链下信息（如API、LLM输出），并将感知结果纳入决策过程。
>
> **(C3) 自主决策（Autonomous Decision-Making）：** 该实体在无需逐笔交易级别的人类批准下，自主决定行动的时机、目标和参数。决策机制包含非确定性成分——即在相同的环境输入下，不同运行可能产生不同的行为输出。
>
> **(C4) 适应性（Adaptiveness）：** 该实体能够根据环境反馈修改自身行为。修改可体现为参数调整（如改变交易规模）、策略切换（如从套利切换为做市）或目标重设。

## 2. 边界判定：什么不是 AI Agent

| 实体类型 | C1 链上执行 | C2 环境感知 | C3 自主决策 | C4 适应性 | 判定 |
|----------|:----------:|:----------:|:----------:|:--------:|------|
| 人类用户 (通过钱包手动操作) | ✓ | ✓ | ✗ (人类审批) | ✓ | **非 Agent** |
| 确定性脚本 (cron job) | ✓ | ✗ (固定触发) | ✗ (确定性) | ✗ | **非 Agent** |
| 简单交易 Bot (网格/DCA) | ✓ | ✓ | ✗ (确定性规则) | ✗ | **非 Agent** |
| 智能合约 (AMM/Router) | ✗ (被动调用) | ✗ | ✗ | ✗ | **非 Agent** |
| 交易所热钱包 | ✓ | ✓ | ✗ (人工+脚本) | ✗ | **非 Agent** |
| MEV Searcher | ✓ | ✓ | ✓ | ✓ | **是 Agent** |
| RL 交易 Agent | ✓ | ✓ | ✓ | ✓ | **是 Agent** |
| DeFi 管理 Agent (Autonolas) | ✓ | ✓ | ✓ | ✓ | **是 Agent** |
| LLM 驱动 Agent (AI16Z) | ✓ | ✓ | ✓ | ✓ | **是 Agent** |
| 自治 DAO Agent | ✓ | ✓ | ✓ (集体自主) | ✓ | **是 Agent** |
| 跨链桥 Relayer | ✓ | ✓ | 部分 | 部分 | **边界案例** |

## 3. 操作化判定标准（链上可观测）

四个条件均无法直接从链上数据"看到"，但可通过以下代理指标推断：

### C1 验证：链上执行能力
- `eth_getCode(address) == "0x"` → 是 EOA（非合约）
- 该地址有主动发起的交易（`from == address`）

### C2 验证：环境感知
- 交易参数对环境变量的条件依赖：
  - `calldata_entropy > 0`（调用数据随环境变化）
  - 交易时机与链上事件相关（如价格变动后触发交易）

### C3 验证：自主决策
- **非确定性判定**：相似环境条件下行为存在变异
  - `tx_interval_cv > threshold_cv`（交易间隔变异系数 > 阈值）
  - `calldata_entropy > threshold_entropy`（调用数据在相似条件下不完全一致）
- **无人类审批**：
  - `active_hour_entropy > threshold_hour`（活动时间分布均匀，无昼夜节律 → 非人类操作）
  - `burst_ratio > threshold_burst`（存在密集交易突发，人类无法手动完成）

### C4 验证：适应性
- 行为参数的时序变化（非随机漂移）：
  - Gas 策略的时序自相关结构变化
  - 交易规模或目标合约集合随时间系统性演变
  - 策略切换点可被检测到（如从探索转向利用）

## 4. 与 Paper 0 分类体系的映射

Paper 0 的 8 类分类中，**满足 Agent 定义（C1-C4）的有 5 类**：

| 类别 | C1 | C2 | C3 | C4 | Agent? |
|------|:--:|:--:|:--:|:--:|:------:|
| 1. 确定性脚本 | ✓ | ✗ | ✗ | ✗ | 否 |
| 2. 简单交易Bot | ✓ | ✓ | ✗ | ✗ | 否 |
| 3. MEV Searcher | ✓ | ✓ | ✓ | ✓ | **是** |
| 4. 跨链桥Agent | ✓ | ✓ | 部分 | 部分 | 边界 |
| 5. RL交易Agent | ✓ | ✓ | ✓ | ✓ | **是** |
| 6. DeFi管理Agent | ✓ | ✓ | ✓ | ✓ | **是** |
| 7. LLM驱动Agent | ✓ | ✓ | ✓ | ✓ | **是** |
| 8. 自治DAO Agent | ✓ | ✓ | ✓ | ✓ | **是** |

分类体系涵盖了 Agent 和非 Agent 实体，定义则从中划出明确的边界线。
**边界线位于自主性等级 ADAPTIVE（等级2）及以上** —— NONE 和 REACTIVE 不满足 C3+C4。

## 5. 各 Paper 如何引用本定义

### Paper 1 (链上 AI Agent 识别)
> "我们采用 Paper 0 中建立的形式化定义（定义1），将满足 C1-C4 四个条件的
> EOA 地址识别为 AI Agent。具体地，本文的特征工程将 C2-C4 操作化为 23 维
> 链上可观测特征，分类器的任务是判断一个地址是否同时满足全部四个条件。"

### Paper 2 (Agent 工具接口安全)
> "根据 Paper 0 的定义，AI Agent（满足 C1-C4）通过工具接口扩展其环境感知（C2）
> 和链上执行（C1）能力。工具接口是 Agent 定义中 C1 和 C2 的关键实现层，
> 因此其安全漏洞直接威胁 Agent 的正确性和安全性。"

### Paper 3 (AI Agent Sybil 攻击)
> "传统 Sybil 检测针对的是确定性脚本和简单 Bot（不满足 C3-C4 的实体）。
> 本文研究 AI Agent（满足 C1-C4，特别是 C3 自主决策和 C4 适应性）
> 利用其适应能力规避现有检测器。Agent 的 C4（适应性）恰恰是使其能够
> 模仿人类行为以规避检测的核心能力。"

## 6. 与经典定义的关系

| 经典定义 | 我们的对应 | 差异 |
|----------|-----------|------|
| Russell & Norvig: "感知+行动" | C1 + C2 | 我们增加了 C3(自主) + C4(适应)，因为 R&N 定义过于宽泛（温度计也满足） |
| Wooldridge & Jennings: 自主+社交+反应+主动 | C3≈自主, C2≈反应, C4≈主动 | 我们不要求社交能力；增加链上执行(C1)作为 Web3 特有条件 |
| Franklin & Graesser: 属性子集分类 | 我们的 C1-C4 是最小必要属性集 | 他们罗列属性组合，我们给出必要充分条件 |
| Maes: 通过可观测行为定义 | 第3节操作化标准 | 完全一致的哲学立场，我们具体化到了链上指标 |
